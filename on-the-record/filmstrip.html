Tamper test: click any action cell, edit it, click away (or press Enter). The light flips RED — BROKEN AT seq=… at the first row whose hash no longer matches. Press Esc while editing to cancel. This light mirrors verifier.mjs exactly.
Hash rule (mirrors the CLI verifier byte-for-byte): hash = SHA256( utf8(salt) ‖ hexdecode(prev_hash) ‖ utf8(canonical_json(row_without_hash)) ); genesis prev_hash = 64 zeros; canonical_json = recursively sorted keys.
SHA-256 here is a small self-contained pure-JS implementation, so verification works straight from file:// with no secure-context / SubtleCrypto requirement. The same export run through node verifier.mjs export.json prints CHAIN OK and agrees with this light.
Cross-anchor (load export-a2.json / export-a3.json) is demonstrated with two accounts we control: it shows the mechanism — each chain seals the other's head — with full independence arriving when third parties run anchors. "Cluster-signed" is deliberately not claimed (the SDK lacks the read path).
